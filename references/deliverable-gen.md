# 📝 交付物生成 Agent（Deliverable Generator）

## 定位

按人确认的结论 + 模板优先级规则，生成全套报价交付物。生成完毕后执行一致性检查和知识回写。

- [ ] `out/分析/README.md` 存在且内容非空
- [ ] `out/分析/客户模板清单.md` 存在
- [ ] `out/分析/客户交付物需求清单.xlsx` 存在
- [ ] `out/分析/特殊特性清单.xlsx` 存在

## 执行步骤

### C1. 读取客户交付物需求清单

🛑 **必须从 `out/分析/客户交付物需求清单.xlsx` 读取，不可自行编造交付物清单。**

该清单由文档解析阶段（A6）从客户文件中自动提取并经用户确认，包含：
- 客户要求的每项交付物
- AI 可生成性判定（✅ / ⚠️ / ❌）
- 对应的客户模板（如有）

**处理逻辑**：
- **✅ 可生成** → 正常生成
- **⚠️ 部分生成** → 生成框架/模板，黄色标记人工填写区域
- **❌ 不可生成** → 跳过，确保已在缺失项跟踪表中标为阻塞项
- **不在客户清单中** → 不生成（除非用户额外要求）

生成的交付物 → 存入 `out/交付物/`
分析过程文件 → 存入 `out/分析/`
流程记录 → 存入 `out/记录/`

### C2. 按依赖顺序生成

🛑 **必须遵循 `references/deliverable-order.md` 中定义的依赖顺序。依赖项未完成时，禁止生成下游交付物。**

对每项选中的交付物：

1. **检查模板优先级**（🛑 必须遵循 `references/template-priority.md`）
2. 从前序阶段产出中准备填充内容
3. **走客户模板路径**：读取客户模板 → 定位填写区域 → 注入数据 → 另存为 `out/交付物/{交付物名}.xlsx`
4. **走自主生成路径**：用 openpyxl 按 `references/excel-style.md` 直接创建 → 保存到 `out/交付物/{交付物名}.xlsx`

逐个生成，每个等用户确认。

### C2b. 生成可行性承诺书（FC）

🛑 **本步骤必须在全部客户要求交付物生成完毕后执行，无论客户交付物需求清单中是否列明，均必须生成。**

**数据来源**（从前序产出中自动提取，不需要用户额外输入）：

| 内容模块 | 数据来源文件 |
|---|---|
| 项目基本信息 | `out/分析/README.md` |
| 供应商信息 | `company-profile/` |
| 特殊特性承诺（CC/SC） | `out/分析/特殊特性清单.xlsx` |
| 质量目标承诺 | `out/分析/质量目标表.xlsx` |
| 技术例外项 | `out/分析/例外事项清单.xlsx` |
| 风险声明 | `out/分析/风险提示清单.md` |
| 总体可行性结论 | 用户在 Step 3 提供的工程判断决策结论 |

**生成规则**：

1. **检查客户模板**：如客户提供了可行性承诺书模板，优先使用（`references/template-priority.md` 铁律）
2. **无客户模板时**，按以下结构生成 `.docx` 文件：

```
一、基本信息
  客户、项目号、零件号/名称、RFQ编号、评审日期、供应商名称

二、总体可行性结论
  ☐ Go  ☐ 条件 Go（前提条件：___）  ☐ No-Go（原因：___）

三、特殊特性确认（CC/SC）
  逐条列出每项 CC/SC、承诺 Cpk、管控方案

四、质量目标承诺
  PPM目标、关键尺寸 Cpk 要求、0km 目标

五、制造可行性声明
  工艺路线确认、DFM结论、产能/节拍确认

六、技术例外项
  逐条列出例外项，注明替代方案或偏差申请状态

七、风险声明
  列出主要风险项及管控措施（直接从风险提示清单提取高/中级风险）

八、资源与进度承诺
  CFT团队确认、APQP关键里程碑（SOP日期、PPAP提交日期）

九、签字栏
  工程负责人、质量负责人、销售负责人、管理层（黄色待签）
```

3. **输出路径**：`out/交付物/可行性承诺书.docx`（或按客户模板格式）
4. **黄色标注**：无法从前序产出中自动提取的内容（如 SOP 日期）用黄色标注，提示用户填写
5. **语言**：中文为主，保留客户原文关键术语对照

### C3. 最终一致性检查

全部交付物生成完毕后：

🛑 **必须加载 `references/final-consistency.md` 并逐条执行。不可跳过。**

检查结果写入 `out/记录/最终检查报告.md`。

### C4. 知识回写

一致性检查全部 PASS 后：

🛑 **必须加载 `references/knowledge-writeback.md` 并执行。不可跳过。**

---

## 核心原则

- **模板优先级铁律**：客户模板 > 自建。见 `references/template-priority.md`
- **依赖顺序约束**：见 `references/deliverable-order.md`
- **多条件必须分行**：同一参数在不同条件下有不同要求值时，每个条件独立一行
- **中文输出**：保留关键英文/德文原文对照
- **公司数据自动填**：有 company-profile/ 则自动读取填充，无则输出黄色待填模板
