# 交付物依赖顺序

交付物生成 Agent 必须按以下依赖关系决定生成顺序。**依赖项未完成时，禁止生成下游交付物。**

## 依赖图

```
README(BOM+数据) ─┬→ Q1 成本分解
                  ├→ L1 工程特性清单 ─┬→ L2 零部件特性清单 ─→ QFD 质量屋矩阵
                  │                   └→ Q2 开发计划表
                  └→ Q6 补充材料清单

Q3 团队表 ────────→ Q5 职责分配矩阵（RASI）

A5 特殊特性清单 ──→ Q4 例外事项清单
```

## 生成顺序（推荐执行顺序）

### 第一批（无依赖，可并行）

| 编号 | 交付物 | 依赖 |
|------|--------|------|
| Q3 | 团队表 | 无（需 company-profile/SDT团队表） |
| A5 | 特殊特性清单 | 无（Phase B 已生成，可重新生成） |
| A6 | 质量目标表 | 无（Phase B 已生成，可重新生成） |
| MIS | 缺失项跟踪表 | 无（Phase B 已生成，可重新生成） |

### 第二批（依赖第一批或 README）

| 编号 | 交付物 | 依赖 |
|------|--------|------|
| Q1 | 成本分解 | README BOM |
| Q4 | 例外事项清单 | A5 特殊特性清单 |
| Q5 | 职责分配矩阵（RASI） | Q3 团队表 |
| Q6 | 补充材料清单 | README |
| L1 | 工程特性清单 | README |

### 第三批（依赖第二批）

| 编号 | 交付物 | 依赖 |
|------|--------|------|
| L2 | 零部件特性清单 | L1 |
| Q2 | 开发计划表 | L1 |

### 第四批（依赖第三批）

| 编号 | 交付物 | 依赖 |
|------|--------|------|
| QFD | 质量屋矩阵 | L1 + L2 |

### 第五批（汇总层，依赖全部前序产出）

| 编号 | 交付物 | 依赖 |
|------|--------|------|
| FC | 可行性承诺书 | 全部前序交付物 + 用户工程判断决策结论 |

> **注意**：可行性承诺书是 Phase 0 的标志性产出，无论客户交付物需求清单中是否列明，均必须生成。

## L1/L2/QFD 特别规则

- **数据来源**：`out/分析/README.md`（Phase A 产出），不回读原始 PDF
- **L1 编号**：顺序 1, 2, 3...；类别 C(关键) / A(一般)
- **L2 编号**：PC-01, PC-02...；按零部件分组排列；每条 L2 关联父 L1 ID
- **L2 BOM 推断**：从测试对象/接口描述/材料要求/设备分类表反推子零件
- **QFD 关联强度**：◎=9（强）/ ○=3（中）/ △=1（弱）
- **多条件必须分行**：同一参数在不同条件下有不同要求值时，每个条件独立一行
