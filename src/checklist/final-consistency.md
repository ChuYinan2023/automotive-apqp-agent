# 最终一致性检查

## 执行时机

全部交付物生成完毕后，由交付物生成 Agent 加载本文件并逐条执行。

## 执行规则

🛑 **逐条检查，每条必须明确判定 PASS 或 FAIL。**
🛑 **任何一条 FAIL → 停止 → 输出失败项及具体差异 → 等用户决定修复方案 → 修复后重新检查。**
🛑 **不得跳过任何检查项。不得将未验证项标为 PASS。**
🛑 **全部 PASS 之前，禁止触发知识回写。**

## 检查项

### 1. 数字一致性 — PASS / FAIL

特殊特性清单中的要求值 = 例外事项清单中引用的同一参数值 = 补充材料清单中的验收标准值。

**检查方法**：提取三份文档中所有共同参数，逐条比对数值。列出任何不一致项。

### 2. 特性覆盖 — PASS / FAIL

每项 CC（Critical Characteristic）和 SC（Significant Characteristic）在例外事项清单中都有对应行。

**检查方法**：取特殊特性清单中所有 CC/SC 编号，在例外事项清单中逐条查找。列出任何未覆盖项。

### 3. 人员一致 — PASS / FAIL

团队表（Q3）中的人名 = 职责分配矩阵（Q5 RASI）的列头人名。

**检查方法**：提取两份文档中的人名列表，双向比对。列出任何不一致项。

### 4. 测试覆盖 — PASS / FAIL

补充材料清单（Q6）中的测试项覆盖所有需验证的特性。

**检查方法**：从特殊特性清单中提取需验证的特性，在补充材料清单中逐条查找对应测试项。列出任何未覆盖项。

### 5. BOM 一致 — PASS / FAIL

成本分解（Q1）中的 BOM = README 中提取的 BOM。

**检查方法**：比对两处的子零件清单、材料、数量。列出任何差异。

### 6. 模板合规 — PASS / FAIL

所有在 `out/分析/客户模板清单.md` 中标注有客户模板的交付物，是否实际使用了客户模板格式生成。

**检查方法**：逐个检查对应交付物的 sheet 名、列头是否与客户模板一致。列出任何违规项。

### 7. 嵌入完整性 — PASS / FAIL

递归提取层级统计（每层提取数 / 有效文件 / 已删除链接），确认无遗漏。

**检查方法**：读取 `out/_嵌入文件/` 目录，核对与文档解析阶段的统计报告一致。列出任何差异。

### 8. 缺失项状态 — PASS / FAIL

所有"最高"优先级缺失项在缺失项跟踪表中已有处理方案（向客户要/自行购买/内部收集/用户已确认接受）。

**检查方法**：筛选缺失项跟踪表中优先级为"最高"的行，检查每行是否有处理方案。列出任何无方案项。

## 输出

检查结果写入 `out/记录/最终检查报告.md`，格式：

```markdown
# 最终一致性检查报告

| # | 检查项 | 结果 | 备注 |
|---|--------|------|------|
| 1 | 数字一致性 | PASS/FAIL | 差异详情（如有） |
| 2 | 特性覆盖 | PASS/FAIL | 未覆盖项（如有） |
| ... | ... | ... | ... |

总结：X/8 PASS
```
