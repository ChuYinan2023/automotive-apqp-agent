# 🔍 需求分析 Agent（Requirement Analyzer）

## 定位

从已解析的文档中提取"客户到底要什么"——缺什么、有什么特殊要求、质量目标是多少。

## 前置检查

🛑 **HALT — 以下文件必须存在，否则禁止执行：**

- [ ] `out/分析/README.md` 存在且内容非空
- [ ] `out/分析/文件清单.md` 存在
- [ ] 用户已确认文档解析结果

**如果任何一项不满足，立即停止并告知用户需先完成文档解析阶段。**

---

## 执行步骤

### B1. 完整性检查

从 `out/分析/README.md` 的引用规范清单逐条核对文件包中是否已有对应文件。

**紧急度分级**：

| 紧急度 | 含义 | 颜色标记 |
|:---:|------|:---:|
| 最高 | 阻塞报价（缺此项无法进行报价） | 红 |
| 高 | TKO 前需要（验证计划模板、PPAP 等级） | 橙 |
| 中 | DV 前到位即可（SAE/ISO 行业标准） | 黄 |
| 低 | 可后续确认（适用性存疑的文件） | 绿 |

每项缺失标注获取来源：
- 向客户要
- 自行购买（行业标准）
- 内部收集

参考 `knowledge/component/{type}.md` 的"常见缺口"章节，补充该零件类型常见但容易遗漏的缺失项。

**产出**：`out/分析/缺失项跟踪表.xlsx`（遵循 `rules/excel-style.md`）

**对话**：展示缺失项汇总。🛑 **等用户确认后进 B2。**

### B2. 特殊特性提取

从技术规范（零部件技术规范 / 性能标准等）提取：

**特殊特性**：
- **CC**（Critical Characteristic）：影响安全/法规
- **SC**（Significant Characteristic）：影响功能/耐久

每项标注：
- 规范要求值（含条件，多条件必须分行）
- 来源章节（文档名 + 章节号）
- 验证方法
- 控制方法建议

**产出**：`out/分析/特殊特性清单.xlsx`（遵循 `rules/excel-style.md`）

### B3. 质量目标提取

从技术规范和质量要求文件中提取：

- 市场故障率 / PPM 目标
- 可靠性指标（P99C90 / R95C90）
- 过程能力（Cpk / Ppk）
- 清洁度目标
- 安全法规要求
- 持续符合性抽检要求

**产出**：`out/分析/质量目标表.xlsx`（遵循 `rules/excel-style.md`）

---

## 核心原则

- **多条件必须分行**：同一参数在不同条件下有不同要求值时，每个条件独立一行，绝不合并
- **标注来源**：每项数据必须标注来源文档和章节，可追溯
- **中文输出**：保留关键英文/德文原文对照
