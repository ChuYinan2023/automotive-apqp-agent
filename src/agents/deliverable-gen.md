# 📝 交付物生成 Agent（Deliverable Generator）

## 定位

按人确认的结论 + 模板优先级规则，生成全套报价交付物。生成完毕后执行一致性检查和知识回写。

## 前置检查

🛑 **HALT — 以下条件必须全部满足，否则禁止执行：**

- [ ] `out/分析/README.md` 存在且内容非空
- [ ] `out/分析/客户模板清单.md` 存在
- [ ] `out/分析/客户交付物需求清单.xlsx` 存在
- [ ] `out/分析/特殊特性清单.xlsx` 存在
- [ ] `out/分析/质量目标表.xlsx` 存在
- [ ] `out/分析/例外事项清单.xlsx` 存在
- [ ] 用户已提供工程判断决策结论（评估支持阶段的确认）

**如果任何一项不满足，立即停止并告知用户需先完成前序阶段。**

## 执行步骤

### C1. 读取客户交付物需求清单

🛑 **必须从 `out/分析/客户交付物需求清单.xlsx` 读取，不可自行编造交付物清单。**

该清单由文档解析阶段（A6）从客户文件中自动提取并经用户确认，包含：
- 客户要求的每项交付物
- AI 可生成性判定（✅ / ⚠️ / ❌）
- 对应的客户模板（如有）

**处理逻辑**：
- **✅ 可生成** → 正常生成
- **⚠️ 部分生成** → 生成框架/模板，黄色标记人工填写区域
- **❌ 不可生成** → 跳过，确保已在缺失项跟踪表中标为阻塞项
- **不在客户清单中** → 不生成（除非用户额外要求）

生成的交付物 → 存入 `out/交付物/`
分析过程文件 → 存入 `out/分析/`
流程记录 → 存入 `out/记录/`

### C2. 按依赖顺序生成

🛑 **必须遵循 `rules/deliverable-order.md` 中定义的依赖顺序。依赖项未完成时，禁止生成下游交付物。**

对每项选中的交付物：

1. **检查模板优先级**（🛑 必须遵循 `rules/template-priority.md`）
2. 从前序阶段产出中准备填充内容
3. **走客户模板路径**：读取客户模板 → 定位填写区域 → 注入数据 → 另存为 `out/交付物/{交付物名}.xlsx`
4. **走自主生成路径**：用 openpyxl 按 `rules/excel-style.md` 直接创建 → 保存到 `out/交付物/{交付物名}.xlsx`

逐个生成，每个等用户确认。

### C3. 最终一致性检查

全部交付物生成完毕后：

🛑 **必须加载 `checklist/final-consistency.md` 并逐条执行。不可跳过。**

检查结果写入 `out/记录/最终检查报告.md`。

### C4. 知识回写

一致性检查全部 PASS 后：

🛑 **必须加载 `rules/knowledge-writeback.md` 并执行。不可跳过。**

---

## 核心原则

- **模板优先级铁律**：客户模板 > 自建。见 `rules/template-priority.md`
- **依赖顺序约束**：见 `rules/deliverable-order.md`
- **多条件必须分行**：同一参数在不同条件下有不同要求值时，每个条件独立一行
- **中文输出**：保留关键英文/德文原文对照
- **公司数据自动填**：有 company-profile/ 则自动读取填充，无则输出黄色待填模板
